Design Section 3: Configuration & Integration

Environment variables, .illustyle format, and orchestrator wiring

New Environment Variables

.env additions
Variable Required Description
ILLUSTRATION_ENABLED No "true" / "false". Default: "true"
ILLUSTRATION_DEFAULT_STYLE No Global style prompt. Default: "modern tech illustration, clean lines, professional"
ILLUSTRATION_DEFAULT_SIZE No Image size. Default: "1792x1024"
ILLUSTRATION_DEFAULT_QUALITY No "standard" / "hd". Default: "hd"
ILLUSTRATION_DIR No Local storage path. Default: "./data/illustrations"
IMAGE_PROVIDER_PRIORITY No Comma-separated provider order. Default: "dalle,stability,midjourney,openai_compat"
OPENAI_API_KEY If DALL-E used For DALL-E 3 and OpenAI-compatible endpoints
STABILITY_API_KEY If Stability used Stability AI API key
MIDJOURNEY_API_KEY If Midjourney used Midjourney proxy API key
MIDJOURNEY_ENDPOINT If Midjourney used Midjourney proxy base URL

.illustyle Per-Repo Override

Example: tech repo
# .illustyle
# For a Go backend project

style: "isometric 3D diagram of microservices architecture, blue and orange color scheme, dark background"
provider: dalle
quality: hd
size: "1792x1024"
Example: creative repo
# .illustyle
# For an art/creative project

style: "watercolor painting, soft pastel colors, dreamy atmosphere"
provider: midjourney
quality: hd
size: "1024x1024"

Opt-out in .repoignore

.repoignore additions
# Existing repo ignore patterns
internal-tool
test-project-*

# NEW: Disable illustration generation for matching repos
no-illustration

When no-illustration is present, the repo is scanned and content is generated as normal, but illustration generation is skipped entirely.
