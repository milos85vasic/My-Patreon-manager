Design Section 2: Data Model & Database

New models, migrations, and storage

New Model: Illustration

internal/models/illustration.go
type Illustration struct {
    ID                 string    `json:"id"`
    GeneratedContentID string    `json:"generated_content_id"`
    RepositoryID       string    `json:"repository_id"`
    FilePath           string    `json:"file_path"`    // local: ./data/illustrations/abc.png
    ImageURL           string    `json:"image_url"`    // if hosted
    Prompt             string    `json:"prompt"`
    Style              string    `json:"style"`
    ProviderUsed       string    `json:"provider_used"`
    Format             string    `json:"format"`        // png, jpeg, webp
    Size               string    `json:"size"`          // 1792x1024
    ContentHash        string    `json:"content_hash"`  // SHA-256 of image data
    Fingerprint        string    `json:"fingerprint"`   // prompt+style hash for idempotency
    CreatedAt          time.Time `json:"created_at"`
}

Key constraint: One illustration per GeneratedContent. The Fingerprint field ensures idempotency — if the prompt+style haven't changed, the existing illustration is reused.

Database Migration

0002_illustrations.up.sql
CREATE TABLE illustrations (
    id TEXT PRIMARY KEY,
    generated_content_id TEXT NOT NULL,
    repository_id TEXT NOT NULL,
    file_path TEXT NOT NULL,
    image_url TEXT DEFAULT '',
    prompt TEXT NOT NULL,
    style TEXT DEFAULT '',
    provider_used TEXT NOT NULL,
    format TEXT DEFAULT 'png',
    size TEXT DEFAULT '1792x1024',
    content_hash TEXT NOT NULL,
    fingerprint TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (generated_content_id)
        REFERENCES generated_contents(id),
    FOREIGN KEY (repository_id)
        REFERENCES repositories(id)
);

CREATE UNIQUE INDEX
  idx_illustrations_content
  ON illustrations(generated_content_id);

CREATE INDEX
  idx_illustrations_fingerprint
  ON illustrations(fingerprint);
0002_illustrations.down.sql
DROP INDEX IF EXISTS
  idx_illustrations_fingerprint;
DROP INDEX IF EXISTS
  idx_illustrations_content;
DROP TABLE IF EXISTS
  illustrations;

File Storage Layout

Local illustration storage
data/
├── illustrations/
├── abc123.png ← by content hash
├── def456.png

Note: Files are named by their content_hash (SHA-256). This prevents duplicates and makes idempotent re-runs free — same prompt+style = same file, no re-generation.
