Illustration Generation Pipeline

Where illustrations fit in the content pipeline

Current Pipeline (without illustrations)
Repository Scan → Content Generation → Quality Gate → Rendering → Publish to Patreon
New Pipeline (with illustrations)
Repository Scan → Content Generation → Quality Gate → ✨ Illustration Generation → Rendering → Publish to Patreon

Illustration Step Detail

What happens inside the illustration step
1. Check .repoignore
Skip if no-illustration or skip-images is set
2. Load Style Config
Per-repo .illustyle or global default
3. Extract Context
Title, topics, language, description → prompt
4. Build Image Prompt
Context + style instructions → final prompt
5. Call Image Provider
DALL-E / Stability / Midjourney / compatible
6. Store & Embed
Save image locally, inject into markdown body

Opt-out Mechanism

.repoignore (existing file)
# Existing ignore patterns
internal-secret-repo
test-*

# NEW: Disable illustration generation
no-illustration

Simple directive in existing file

.illustyle (new per-repo file)
# Illustration style for this repo
style: "watercolor technical diagram"
provider: stability
quality: hd
size: 1792x1024

# Or disable entirely
disabled: true

Rich per-repo override config
