Key Design Decision 1: Integration Point

Where should illustration generation happen in the content pipeline?

A

During Content Generation

Generate illustrations as part of LLM content creation (same prompt/call)

Pros

  • Perfect context awareness
  • Single operation
  • Coherent article+image

Cons

  • Higher token/API cost
  • Longer generation time
  • LLM may not be best at images
B

Post-Content Generation

Generate illustrations after text content is complete (separate pipeline)

Pros

  • Specialized image models
  • Parallel processing
  • Better quality control

Cons

  • Context extraction needed
  • Two-step process
  • Potential mismatch
C

Hybrid Approach

LLM generates image prompts during content creation, separate service renders images

Pros

  • Best of both worlds
  • Context-aware prompts
  • Specialized rendering

Cons

  • Most complex
  • Multiple API calls
  • Prompt quality critical

Key Design Decision 3: Storage & Management

How should generated illustrations be stored and managed?

Option X: Database + Local Files
Structure:
• Database: image metadata, prompts, associations
• Local files: images in user/illustrations/
• References: article_id → image_path

Pros: Simple, fast, no external dependencies
Cons: Manual backup, scaling issues
Option Y: Cloud Storage (S3/R2)
Structure:
• Database: image metadata + cloud URLs
• Cloud: images in bucket with CDN
• References: article_id → cloud_url

Pros: Scalable, durable, CDN delivery
Cons: Additional cost, external dependency

Key Design Decision 4: Brand Consistency

How to ensure consistent "enterprise-grade" visual style?

A

Style Prompts

Add consistent style instructions to every image prompt

Example: "in a modern tech illustration style, clean lines, blue/gray color scheme"
B

Reference Images

Use image-to-image generation with reference style images

Requires: Base style images, more complex generation
C

Custom Fine-tuned Model

Fine-tune a model on approved illustration style

Most consistent, but expensive/complex
